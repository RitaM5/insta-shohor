Local storage এ ডেটা স্থায়ীভাবে থাকে এবং object এর মতো করেও local storage এর মধ্যে ডেটা set,get,remove করা যায়। ব্রাউজারের বিভিন্ন ট‍্যাবে,উইন্ডোর মধ্যে local storage এর ডেটা শেয়ার করা যায় এবং শেয়ার করার কারনে local storage এ ডেটা পরিবর্তন করা যায়। সেই পরিবর্তিত ডেটা অন্য সব ব্রাউজারের ট‍্যাবে চলে যায়।অর্থ‍্যাৎ local storage এর ডেটা remove না করলে একদম স্থায়ী থেকে যায়। local storage এর ডেটা বা key ,value উভয়ই string অবস্থায় থাকে এবং object অনুযায়ীও কাজ করে।

অন‍্যদিকে,session storage এর মধ্যে ডেটা স্থায়ীভাবে থাকে না। local storage এর অবজেক্টের তুলনায় session storage এর object কম use হয়।session storage শুধুমাত্র ব্রাউজারের বর্তমান ট‍্যাবে বিদ‍্যমান থাকে এবং একই page এর ডেটা ভিন্ন ব্রাউজারের ট‍্যাবে open করলে নতুন storage চলে আসে এবং নতুন করে ডেটা set করতে হয়।আবার ব্রাউজার রিফ্রেশ করলে ডেটা থাকে কিন্তু ব্রাউজার বন্ধ বা আপডেট করলে ডেটা মুছে যায়।

একটি জাভাস্ক্রিপ্ট ফাইল এর কোড রান করলে এটা প্রথম থেকে শেষ পর্যন্ত পুরোটা রিড করে। প্রত্যেকটা ইভেন্টকে একটা queue এর ভিতরে ঢুকায়। এই queue থেকে একটা একটা করে প্রত্যেকটা ইভেন্টকে লুপ করে হ্যান্ডেল করাটাকে ইভেন্ট লুপ বলে। queue তে থাকা ইভেন্টকে হ্যান্ডেল করার জন্য প্রয়োজনীয় ফাংশন stack মধ্যে রাখা হয়। নেস্টেড ফাংশনের ক্ষেত্রে একটা একটা করে চাইল্ড ফাংশনগুলো কে stack এর ভিতর রাখা হয়। সবথেকে শেষের চাইল্ড ফাংশন টা কে এক্সিকিউট করে সেটার প্যারেন্ট ফাংশন কে কল করা হয়। এভাবে একটা একটা করে ফাংশন গুলোকে execute করা হয় যতক্ষণ না stack খালি হয়ে যায়। সম্পূর্ণ খালি হয়ে যাওয়ার পর queue থেকে পরবর্তী ইভেন্ট কে হ্যান্ডেল করা হয়।
